super(type[,object-or-type])函数

环境：

  • python版本:3.7.4
  • IDE: spyder

MRO列表

MRO，即method resolution order，方法解析顺序。MRO列表即查找成员时搜索全部基类所用的先后顺序组成的列表。
其顺序为：子类先于超类，尊重base的先后顺序。例子见下。

super()函数

super()为python的内置函数，主要用于处理类继承出现的问题。比如初始化时，比如调用超类的方法的时候。
python3.x可以直接使用super().XXX替代python2.x的super(class,self).xxx
在单继承里，可以避免直接显示超类名而提高可维护性。比如：

class Base():
    def __init__(self):
        pass

class A(Base):
    def __init__(self):
        super(A,self).__init__()    #在这里避免了出现Base类名

如果我们想让A继承另一个超类，直接修改继承的类名即可：

#直接修改类名
class A(Another_Base):
    def __init__(self):
        super(A,self).__init__()    #这里就可以避免修改

当然在使用super()函数时可以直接super().__init__(),下面的例子也是如此。
在多重继承时，该方法使菱形图成为可能：

class Base():
    def __init__(self):
        print("Base")

class A(Base):
    def __init__(self):
        super(A,self).__init__()
        print("A")

class B(Base):
    def __init__(self):
        super().__init__()    #使用python3.x的用法
        print("B")

class C(A,B):
    def __init__(self):
        super(C,self).__init__()
        print("C")

d = C()
print('='*20)
print(C.mro())    #获取MRO列表

参数的含义

inst通过属性提供整个MRO，当前类名cls告诉它当前的位置。

所以super()实际上做的差不多是:

def super(cls, inst):
    mro = inst.__class__.mro() # Always the most derived class
    return mro[mro.index(cls) + 1]

实用建议：

#使用**kwds，每次调用都剥离需要的参数
class Shape：
    def __init __(self，shapename，**kwds):
        self.shapename = shapename 
        super().__ init __(**kwds)         

class ColoredShape(Shape):
    def __init __(self，color，**kwds)：
        self。 color = color 
        super().__ init __(**kwds)

cs = ColoredShape(color='red', shapename='circle')
